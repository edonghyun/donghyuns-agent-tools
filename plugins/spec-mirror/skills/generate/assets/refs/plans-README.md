# Plans

Execution plans for tasks that change the implementation. Plans are short-lived: they live here while work is in flight, then move to `decisions/` or are deleted once the change ships.

## Naming convention

`NN-<slug>.md` — mirror the design number when one exists, e.g. `01-oauth-login.md`.

## When to create

- A multi-step task that needs to be paused and resumed.
- Multi-agent collaboration where one agent must hand off to another.
- A change touching code that has `/spec-mirror:scope` output worth attaching.

## Suggested sections

```markdown
# Plan: <task>

**Linked design:** `refs/designs/NN-….md`
**Spec scope:** see `specs/scope/<target>.md` (generated by /spec-mirror:scope)

## Steps
- [ ] step 1 — file/dir
- [ ] step 2 — file/dir

## Out of scope
## Done when
```
